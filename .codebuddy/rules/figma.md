# Figma 设计分析与导出

## 核心能力

### 1. 文件操作与分析
- **文件检查**：获取任何 Figma 文件的完整 JSON 表示
- **组件提取**：列出所有组件、样式和设计令牌
- **资产导出**：批量导出框架、组件或特定节点为 PNG/SVG/PDF
- **版本管理**：访问特定文件版本和分支信息

### 2. 设计系统管理
- **样式审计**：分析颜色使用、排版一致性、间距模式
- **组件分析**：识别未使用的组件、衡量使用模式
- **品牌合规**：检查文件是否符合品牌指南
- **设计令牌提取**：从 Figma 样式生成 CSS/JSON 设计令牌

### 3. 批量资产导出
- **多格式导出**：导出 PNG、SVG、PDF 或 WEBP 格式的资产
- **平台特定尺寸**：为 iOS/Android 生成 @1x、@2x、@3x 资产
- **有组织的输出**：按格式或平台自动文件夹组织
- **客户包**：带文档的完整可交付包

### 4. 无障碍与质量分析
- **对比度检查**：验证 WCAG 颜色对比度要求
- **字体大小分析**：确保可读的排版比例
- **交互元素尺寸**：检查触摸目标要求
- **焦点状态验证**：验证键盘导航模式

## 工作流模式

### 设计系统审计工作流
1. 提取文件数据 → 获取组件、样式和结构
2. 分析一致性 → 检查样式变化和未使用元素
3. 生成报告 → 创建详细发现和建议
4. 手动实施 → 使用发现指导设计改进

### 资产导出工作流
1. 识别导出目标 → 指定框架、组件或节点
2. 配置导出设置 → 设置格式、大小和命名约定
3. 批量处理 → 同时导出多个资产
4. 组织输出 → 为交付或实施构建文件

## 限制
- **只读操作**：通过 REST API 对 Figma 文件进行只读访问
- 不能修改现有文件、创建新设计、批量更新或实时协作
