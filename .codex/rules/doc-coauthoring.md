# 文档协作工作流

## 适用场景
- PRD
- 技术方案
- 决策文档
- RFC
- 重要提案或规格说明

## 三阶段流程

### 1. 上下文收集
- 明确文档类型、目标读者、预期影响、模板和约束
- 鼓励一次性提供背景、讨论记录、限制条件和依赖信息
- 根据缺口继续追问，直到可以讨论边界和权衡

### 2. 结构与优化
- 按章节推进，而不是一次性写完整篇
- 每节先澄清，再列出要点候选，再筛选合并，最后起草
- 迭代修改时尽量做局部编辑，不重复整篇重写

### 3. 读者测试
- 预测读者会提出的 5-10 个问题
- 用没有上下文的人或新对话检验文档是否清楚
- 修复含糊、矛盾和默认前提过多的问题

## 质量标准
- 结构清晰
- 跨章节一致
- 无明显废话
- 对目标读者足够自洽
