## 1. 技术栈决策

基于PRD中的“最小可行产品（MVP）”定位和性能要求（生成时间<30秒），遵循知识库4.1 技术栈决策框架的指导，选择以下技术栈以实现快速开发、清晰分离和易于部署：

**后端 (API Server):**

- **框架:** Nest.js (Node.js)。选择理由：轻量、现代、异步中间件模型适合IO密集的AI API调用场景。
- **数据库:** PostgreSQL (或云托管服务如Supabase/Aiven)。选择理由：关系型数据模型适合管理用户、故事、生成记录等结构化数据，且可靠性高。
- **ORM:** Prisma。用于安全、类型化的数据库操作。
- **AI服务集成:** 调用第三方AI API（如OpenAI的ChatGPT用于文本生成，MinMax/GLM API用于图像生成）。
- **文件存储:** 云存储服务（如七牛云）用于存储生成的绘本图片和PDF。
- **部署:** 考虑无服务器部署（参考12.2 部署到类 Vercel 平台）或容器化部署（参考14.3.1 应用商店与 Docker 基础）。

**前端 (Web App):**

- **框架:** Vue 3 (Composition API)。选择理由：渐进式、响应式系统优秀，生态丰富，适合快速构建交互复杂的单页面应用。
- **构建工具:** Vite。选择理由：开发体验快，热更新迅速。
- **UI 组件库:** PC端和移动端使用Tailwind CSS实现响应式, 自行实现UI。
- **HTTP 客户端:** Axios。用于与后端API通信。
- **部署:** 静态资源部署到Webify（参考12.1 部署到 EdgeOne Pages）。

**开发与运维:**

- **版本控制:** Git (参考11.1 为什么需要 Git)。
- **API 测试:** ApiPost。
- **环境管理:** 使用`.env`文件管理密钥（参考8.1 密钥管理与环境变量）。

## 2. 系统架构设计

采用典型的前后端分离架构（参考4.5 前后端分离概念）：

- **前端 (Vue3 SPA):** 运行在用户浏览器中，负责所有用户交互（输入、预览、编辑），通过HTTP API与后端通信。
- **后端 (Nest.js API Server):** 提供RESTful API，处理业务逻辑、用户认证、数据库操作，并集成第三方AI服务。
- **第三方服务:** AI文本/图像生成API、云存储、数据库服务。
